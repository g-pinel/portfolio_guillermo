About me

I am a bioinformatician with a strong interest in human health, software development and project management. Experienced in R and analysis of omics data, always eager to learn the tools and write the code needed to research and fight disease.

As a postdoctoral researcher in Christiana Ruhrberg’s lab at UCL, I analyse bulk and single-cell RNAseq data from patients with peripheral artery disease and from mouse models that reproduce this disease. This work is funded by the Leducq Foundation and by the British Heart Foundation.

In this website, you can find links to my repositories, which contain code to easily run and automate various transcriptomic analyses, source code from my Shiny apps, and others. On the publications page you can find information about my scientific articles.


Repositories

runDESeq2

https://github.com/g-pinel/runDESeq2

Bulk RNA-seq experiments often include many samples belonging to multiple experimental groups and involve performing multiple comparisons between different subsets of these samples.

The functions in this repository allow you to run one or multiple pairwise differential expression analyses using the R DESeq2 package.

runEnrichmentAnalyses

https://github.com/g-pinel/runEnrichmentAnalyses







After performing differential expression analyses to obtain a list of differentially-expressed genes (DEGs), it is common to run enrichment analyses to identify groups of genes that participate in the same process or pathway and are enriched in one of the groups being compared.

The functions in this repository allow you to easily run enrichment analyses in R against the Gene Ontology, KEGG, Wiki Pathways and Reactome databases using DESeq2’s DeseqDataSet (DDS) objects as inputs.

autoGSEApreranked

https://github.com/g-pinel/autoGSEApreranked

GSEA (Gene Set Enrichment Analysis) is a software used to determine whether a group of genes shows differences between two biological states (e.g. treatment versus control, high versus low drug dose). In this type of analysis, all genes expressed by the input samples are sorted from those most enriched in one condition to those most enriched in the other, generating a ranked list. Then specific groups of genes are tested against this ranked list to identify those gene sets that are enriched in one of the biological states.

However, standard GSEA analyses do not support controlling for covariates like sex and age. To overcome this, you can supply GSEA with your own ranked list of genes (GSEA Preranked). Differential expression analysis packages like R’s DESeq2, allow to control for covariates, and the test statistic that they output can be used as the ranking metric for the resulting genes.

The scripts in this repository allows you to easily generate the RNK files needed to run GSEA preranked by supplying DESeq2 objects (as RDS files) or results tables (as CSV files). You can do this in batch for multiple input files. In addition, scripts for running one or multiple GSEA preranked analyses with a single command is also provided. You can use the RNK files obtained with the code in this repository, or provide your own.

You can find more information about GSEA here.

qPCR analyzer

https://github.com/g-pinel/qPCR-analyzer

Process your qRT-PCR data generated using the QuantStudio software with a single command. Simply provide your QuantStudio output excel files to obtain CTs, delta CTs, and levels (2-ΔCt).

Both XLS and XLSX supported, as well as combining multiple files from the same project.
